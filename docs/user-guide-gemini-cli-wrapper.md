# Gemini CLI WebUI Wrapper - User Guide

Gemini CLI WebUI Wrapper는 SSH 환경에서 Tmux를 통해 실행되는 **Gemini CLI**를 웹 브라우저에서 편리하게 사용할 수 있도록 돕는 강력하고 가벼운 인터페이스입니다.

이 가이드는 프로젝트의 초기 설정부터 실행, 그리고 최신 기능(파일 뷰어, 자동 동기화 등) 활용 방법까지 상세히 안내합니다.

---

## 1. 사전 준비 (Prerequisites)

이 래퍼를 사용하려면 원격 서버(또는 로컬 머신)에 다음 환경이 준비되어 있어야 합니다.
* **Node.js**: 백엔드 서버(Express, Socket.io) 실행을 위해 필요합니다.
* **Tmux**: 세션 유지 및 터미널 분할을 위한 핵심 유틸리티입니다.
* **Gemini CLI**: 실제 AI 어시스턴트 역할을 수행하는 원본 CLI 도구입니다.

---

## 2. 초기 설정 및 실행 (Setup & Run)

### 단계 1: Tmux 환경 최적화
웹 브라우저에서 완벽한 터미널 경험(마우스 지원, 클립보드 연동, 256 색상 지원, 뷰어 명령어 연동)을 누리려면 Tmux 설정이 필수입니다. 프로젝트에 포함된 자동 설정 스크립트를 사용하세요.

```bash
cd gemini-cli-wrapper
./setup-tmux.sh
```
* **기능**: 사용자의 기존 `~/.tmux.conf`를 덮어쓰지 않고 누락된 필수 옵션만 추가합니다. 또한 터미널 명령어 기반 브라우저 뷰어 호출(`view <file>`) 기능을 셸 환경에 주입하고, 세션을 15분마다 자동 백업/복구하는 `tmux-resurrect` 및 `tmux-continuum`을 설치합니다.
* **적용**: 스크립트 실행 후 Tmux 서버를 완전히 껐다 켜는 것(`tmux kill-server`)을 권장합니다.

### 단계 2: 백엔드 서버 실행
동봉된 `run.sh` 스크립트를 실행하여 백엔드 서버를 구동합니다. (최초 실행 시 `npm install`이 자동으로 수행됩니다.)

```bash
./run.sh
```
* 서버는 기본적으로 `127.0.0.1:5001` (로컬호스트)에서만 연결을 대기합니다. 외부 접근을 차단하여 보안을 강화했습니다.

---

## 3. 웹 브라우저 접속 (Connecting)

보안을 위해 SSH 포트 포워딩(Local Port Forwarding)을 사용하여 로컬 PC에서 원격 서버의 WebUI에 안전하게 접속합니다.

1. **로컬 PC 터미널**에서 다음 명령어를 실행하여 SSH 터널을 생성합니다.
   ```bash
   ssh -L 5001:localhost:5001 <사용자계정>@<서버_주소>
   ```
2. **웹 브라우저**를 열고 다음 주소로 접속합니다.
   👉 `http://localhost:5001`

## 4. 주요 기능 및 사용법 (Features & Usage)

### 4.1. 세션 및 창 관리 (Session Management)
초기 화면에서 서버에 띄워져 있는 모든 Tmux 세션 목록을 확인할 수 있습니다.
* **접속 및 생성**: 기존 세션을 클릭해 접속하거나 `[새 Gemini 세션 생성]` 버튼으로 새 백그라운드 Tmux 세션을 만들고 바로 Gemini CLI를 실행할 수 있습니다.
* **세션명 커스텀**: 터미널 상단의 ✏️ 아이콘을 클릭하여 현재 세션의 이름을 자유롭게 변경할 수 있습니다. 변경된 이름은 **브라우저 탭 제목(Title)**과 동기화됩니다.
* **Keep Alive 설정**: 설정(⚙️) 창에서 `Keep Tmux session alive after Gemini exits` 옵션을 켜면, Gemini CLI를 종료(`/exit`)하더라도 세션이 닫히지 않고 기본 셸(Shell) 상태로 남습니다.
* **화면 분할**: 우측 상단의 ◫(가로 분할), ⬒(세로 분할) 버튼을 눌러 화면을 쉽게 나눕니다. 
* **포커스 강조**: 현재 작업 중인 창(Active Window)은 밝게, 비활성 창은 어둡게(Dim) 표시되어 창 구분이 매우 명확합니다.

### 4.2. 뷰어 및 마크다운 동기화 (File Viewer & Markdown Sync)
웹 UI 내에서 코드를 읽거나 문서를 렌더링 할 수 있는 독립된 뷰어 창을 지원합니다.
* **뷰어 호출**: 
    1. 파일 탐색기에서 파일을 우클릭하여 `View (Split)`을 선택합니다.
    2. 또는 터미널에서 셸 명령어 `view <파일명>`을 입력하면 브라우저에 즉시 팝업으로 렌더링됩니다.
* **실시간 뷰어 (Live Reload)**: 뷰어에 띄운 파일은 서버에서 변경이 감지되면 자동으로 새로고침됩니다.
* **동기화 스크롤 (Sync Scroll)**: 마크다운 파일(.md)을 렌더링할 때 원본 텍스트와 렌더링된 화면 중 한 쪽을 스크롤하면 양방향으로 스크롤이 동기화됩니다.
* **파일 오픈 (Open Modal)**: 뷰어 창 내부의 `[Open]` 버튼을 누르면 서버의 파일을 탐색/정렬하여 다른 파일로 빠르게 교체할 수 있습니다.

### 4.3. 파일 탐색기 (File Explorer)
화면 좌측의 📁 버튼을 눌러 사이드바를 열면 서버의 작업 디렉토리를 탐색할 수 있습니다.
* **터미널 입력**: 파일이나 폴더를 **더블 클릭**하면 터미널 프롬프트에 절대 경로(`@/절대경로/파일.txt `)가 자동 타이핑됩니다.
* **드래그 앤 드롭**: PC의 로컬 파일을 사이드바 위로 드래그하면 현재 보고 있는 디렉토리에 즉시 업로드됩니다.
* **컨텍스트 메뉴**: 항목을 우클릭하여 다운로드, 이름 변경, 삭제, 뷰어로 열기 기능을 사용할 수 있습니다.
* **스마트 자동 동기화**: 터미널에서 `cd` 명령어로 디렉토리를 이동하거나, **Tmux 패널(창)을 전환할 때** 해당 패널이 위치한 현재 경로(PWD)를 탐색기가 자동으로 추적하고 갱신합니다. 수동 동기화는 경로창 우측 📍 핀 버튼을 누릅니다.

### 4.4. 이미지 및 클립보드 (Image & Clipboard)
* **스크린샷 붙여넣기**: 로컬 PC의 이미지를 `Ctrl+V`로 터미널에 붙여넣으면, 현재 PWD 하위의 `screenshots/` 폴더에 즉시 저장됩니다. 저장 후 터미널에 해당 경로가 자동으로 타이핑됩니다.
* **썸네일 바**: 터미널 상단 중앙에 최근 업로드된 5개의 이미지 썸네일이 나타납니다.
    * 클릭: 팝업 모달로 원본 이미지 확대 표시.
    * 체크 후 삽입: 여러 썸네일의 체크박스를 선택한 후 `[선택 삽입]` 버튼을 눌러 프롬프트에 경로를 한 번에 추가합니다.
* **클립보드 히스토리**: 터미널에서 마우스로 텍스트를 드래그하면 브라우저에 자동 복사되며, 좌측 사이드바 하단 `Clipboard History`에 기록이 쌓입니다(최대 5개). 이 항목들을 클릭하여 빠르게 재복사할 수 있습니다.

### 4.5. 기타 설정 (UI & Key Mapping)
* **단축키 커스텀**: 설정(⚙️) 창에서 브라우저의 키보드 입력 이벤트를 감지하여(Record) 특정 키 조합(예: `Cmd+Y`, `Home`)을 커스텀 시퀀스로 매핑하여 원격 서버로 전송할 수 있습니다.
* **테마 및 폰트**: 설정 창에서 Dark/Light 테마를 선택할 수 있으며, 코딩용 폰트(D2Coding 등) 및 크기 조절 상태는 모두 브라우저 `localStorage`에 자동 저장되어 유지됩니다.
